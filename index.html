Sobre a Arquitetura deste Portfólio

Para a construção deste portfólio, optei por uma abordagem focada em agilidade e performance. Em vez de construir uma API Node.js tradicional com banco de dados, desenvolvi uma arquitetura estática.

A página utiliza JavaScript nativo (Vanilla JS) no front-end para buscar e renderizar dinamicamente os dados de um arquivo Markdown (README.md). Esta decisão arquitetural garante um carregamento extremamente rápido, facilidade de atualização direta via texto e custo zero de hospedagem, provando que a solução mais simples muitas vezes é a mais eficiente:


1. Front-end:

HTML5 e CSS3 puro (Vanilla).

2. Motor de Conteúdo:

JavaScript consome este arquivo `README.md` via API Fetch.

3. Renderização:

Utiliza a biblioteca `marked.js` para converter Markdown em HTML dinamicamente.
